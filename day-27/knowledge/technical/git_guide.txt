Git is a distributed version control system used to track changes to source code over time. Common commands include git add to stage changes, git commit to save a snapshot with a message, and git push to upload commits to a remote repository. Branches allow developers to work on features independently before merging into the main branch through a pull request. Commit messages should be clear and descriptive, explaining what changed and why. Before starting new work, developers should pull the latest changes from the main branch to avoid conflicts.
