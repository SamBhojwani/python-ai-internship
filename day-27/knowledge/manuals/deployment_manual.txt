Deployments are triggered automatically when changes are merged into the main branch through the CI/CD pipeline. Before deployment, automated tests and linting checks must pass successfully. The deployment process builds a Docker image, pushes it to the container registry, and updates the running service with zero downtime using a rolling update strategy. Environment specific configuration is injected through environment variables rather than being baked into the image. In case of deployment failure, the pipeline automatically rolls back to the previous stable version.
