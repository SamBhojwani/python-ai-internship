All API requests must include a valid authentication token in the Authorization header. Rate limits are enforced per API key, currently set to 100 requests per minute for standard tier accounts. Responses are returned in JSON format with a consistent structure including a success flag and either data or an error message. API versioning is handled through the URL path, for example /v1/resource, to ensure backward compatibility. Detailed endpoint documentation is available through the interactive Swagger UI at the /docs path.
